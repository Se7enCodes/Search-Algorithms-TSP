TSP Visualizer

Travelling Salesman Problem Solutions using Hill-Climbing Algorithm

The Travelling Salesman Problem (TSP) seeks the shortest route visiting all cities exactly once and returning to the origin. This visualization demonstrates how the Hill-Climbing algorithm iteratively improves solutions by exploring neighboring routes.

The quality of the final solution heavily depends on the initial random configuration. Each restart generates a new starting point, which may lead to better or worse results. This demonstrates why multiple runs with different initial states are crucial for finding optimal solutions—some starting configurations converge to superior local optima than others.

How to Visualize:

  1. Adjust the number of cities using the slider to set problem complexity
  2. Click Start to begin the algorithm and watch it optimize the route
  3. Observe the status badge showing whether the algorithm is improving, searching, or has reached an optimum
  4. Monitor the distance graph below to see how the total distance decreases over iterations
  5. Use Pause to stop and Restart to generate a new random configuration—notice how different starting points lead to different solution qualities (previous runs are shown in lighter gray)
